    ```python
    """Per-session world directory bootstrap (WORLD-02)."""
    from __future__ import annotations
    import shutil
    import uuid
    from pathlib import Path
    from typing import Tuple

    DEFAULT_TEMPLATE_DIR: Path = Path(__file__).parent / "template_world"


    def create_session_world(
        template_dir: Path = DEFAULT_TEMPLATE_DIR,
        sessions_dir: Path | None = None,
    ) -> Tuple[str, Path]:
        """Copy template_dir → sessions_dir/<session_id>/. Return (session_id, abs_session_root).

        Raises FileNotFoundError if template_dir does not exist.
        """
        template_dir = Path(template_dir)
        if not template_dir.is_dir():
            raise FileNotFoundError(f"Template directory not found: {template_dir}")

        if sessions_dir is None:
            sessions_dir = Path.cwd() / "sessions"
        sessions_dir = Path(sessions_dir)
        sessions_dir.mkdir(parents=True, exist_ok=True)

        session_id = uuid.uuid4().hex[:12]
        session_root = (sessions_dir / session_id).resolve()
        shutil.copytree(template_dir, session_root)
        return session_id, session_root
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Template → Session copy | The template is trusted (committed to repo); session copies are mutable per-session sandboxes. |
| User cwd → sessions_dir | Default sessions_dir is `cwd / "sessions"`; user controls cwd, so the location of session data is a user concern, not an attack surface. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | Template directory pollution between sessions | mitigate | `shutil.copytree` deep-copies; Test 5 enforces template stays pristine after session edits |
| T-02-02 | Spoofing | Session ID collision (two sessions get the same dir name) | mitigate | `uuid.uuid4().hex[:12]` — 48 bits of entropy; ~16M sessions before 1% collision; Test 4 verifies distinct IDs across sequential calls. Acceptable for single-user CLI. |
| T-02-03 | Info Disclosure | Session ID leaked to logs identifies the session dir | accept | Local single-user CLI; logs not shipped; benign for v1 |
| T-02-04 | Tampering | Symlinks inside the template are followed by copytree, escaping or duplicating system files | mitigate | Template is committed plain JSON+Markdown — no symlinks. Verify in CI: `find backend_generalist/template_world -type l` returns no rows. (Plan accepts the responsibility but does not gate on this since template is hand-authored from this plan.) |
| T-02-05 | DoS | Massive sessions/ directory accumulates and fills disk | accept | v2 cleanup concern (HARD-03); v1 is a viability spike with a small number of manual sessions |
| T-02-06 | EoP | sessions_dir defaults to cwd, user accidentally creates session in wrong dir | accept | CLI in Plan 04 will print the absolute path at startup (CLI-03), giving the user immediate visibility |
</threat_model>
