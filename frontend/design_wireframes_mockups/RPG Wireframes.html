1 · Lobby / Home

Primary jobs

  • Resume the most recent campaign in one tap
  • Pick a different campaign / character
  • Glance at character status (HP, mana, level, XP-to-next)
  • Start a new campaign or new character
  • Reach settings / auth without crowding play CTAs

Always visible

  • App bar (title + ≡ menu for Settings)
  • No bottom tab nav — navigation between Session ↔ Lobby happens via the top-right ≡ menu
  • "Resume" primary button when an active session exists

Swipe / modal

  • Character switcher: bottom sheet from "Active character" row
  • New campaign: full-screen modal flow

Component inventory (rules-grounded)

  • Resume button → enters Live Session
  • Active character card: name, class (Warrior/Mage/Ranger/Bard), level, HP bar, mana bar (Mage/Bard only), Evasion chip
  • XP-to-next-level bar (thresholds: 100 / 250 / 500 / 1000 …)
  • Pending level-up banner when total XP ≥ next threshold (auto-level on the spot, outside battle)
  • Campaign list rows: title, chapter, time-counter remaining (—1 short, —5 long)
  • Character switcher rail: avatars + class glyphs

UX failures to avoid @ 320px

  • Resume CTA below the fold — hero must be reachable without scrolling.
  • HP/MP bars rendered too thin to read at 1× — minimum 6px tall, 1px stroke.
  • Switch-character action hidden inside settings — surface as ⇅ on the character row.
  • Chapter time counter (the 50-baseline scarcity timer) dropped on this screen — players need to plan rests.
  • Pending level-up shown as a separate inbox screen — must surface inline on the campaign row.
  • Mana bar shown for non-casters — only Mage/Bard.
  • Tap target for campaign rows under 44px tall.

Accessibility / focus

  • Focus order: app bar → Resume → active char card → campaign rows → +New → bottom nav.
  • All tap targets ≥ 44×44pt; campaign rows full-width hit area.
  • HP bar uses both bar AND "28/34" numeric label — never color/length alone.
  • Class is shown as full word in screen reader text ("Warrior"), abbreviated visually ("War").
  • Pending level-up announces with role="status" politely on entry.
  • Contrast: ink #1a on paper #f5f1e8 ≈ 13.4:1 for body; mono labels #6e on paper ≈ 4.7:1 (AA for ≥14px).

2 · Live Session — story-first

Primary jobs

  • Read the unfolding GM/player/system transcript with clear speaker identity
  • Send a player message or "act" — fixed bottom composer
  • Respond to GM-prompted rolls (Attack / Check / Save / Init) inline in chat — no separate dice tray
  • See whether the roll is Normal / Advantage / Disadvantage as a system-set indicator (player never picks)
  • Glance at HP, mana, evasion, conditions in the vital strip
  • Pull the stats row down to expand the Character Sheet inline (chat is pushed down, not overlaid). Two-phase reveal: phase 1 = character header above the stats row · phase 2 = ability scores + combat below it.

Always visible

  • App bar (campaign · chapter · time counter)
  • Vital chips strip: HP · MP · Eva · conditions — doubles as the grab handle for Sheet
  • Composer (text field + primary "act" send)

Swipe / overlay

  • Pull-down handle below the vital strip: drag down (or tap) to expand Character Sheet as an overlay over chat; drag up to dismiss
  • GM-requested roll arrives as a transcript card with: formula, system-set Adv/Norm/Dis indicator (not a player control), and a single [Roll] button. Tap rolls in place; result lands in the next transcript card.
  • Player-initiated free roll: from the composer's "+" menu (rarer); opens roll tray as last-resort. No advantage on free rolls — those route through the GM if relevant.

Component inventory (rules-grounded)

  • Speaker cards: GM · Player · System (rolls, rests, conditions, time-counter changes)
  • Vital strip: HP / MP / Evasion / active conditions — also the Sheet pull-handle
  • In-chat roll prompt: System card with d20 + stat formula + read-only Adv/Norm/Dis indicator (system-set) + [Roll] button. Rolls in place; result lands in next card
  • Crit affordance: shown on the result card when nat-20 lands — "max one set + roll one set + mods once"
  • Mana cost picker (casters): in-chat on Cast prompts — Minor 1 · Major 2 · Mythic 3 (greyed when tier locked)
  • Boss-state banner: when HP=0 in boss → "Blaze of Glory" / "Risk It All (d20, nat 20 = 5 HP)"
  • Rest buttons: Short (HP=hit die+Mig, MP=d4+Wit, t−1) · Long (full, t−5) — surfaced in composer "+" menu or inside the embedded Sheet

UX failures to avoid @ 320px

  • GM-prompted roll cards are the primary roll surface — they must include the formula (d20 + stat vs DC) AND a read-only Adv/Norm/Dis indicator set by the system. Never expose Adv/Dis as a player toggle.
  • Composer collapses behind the iOS keyboard — must use visualViewport / env(safe-area-inset-bottom) and keep input above it.
  • Vital strip sized for desktop — at 320px you have room for 3 chips comfortably, 4 if compact; no 5+.
  • Pull-down handle too small / no visible affordance — must show a 44pt-wide bar plus a "▼ pull" label so it reads as a gesture, not a chevron icon.
  • Sheet rendered as an overlay over chat — must be in flow: stats row stays in place, char header reveals above, ability scores + combat reveal below. Chat is pushed down (and shrinks), never dimmed or covered.
  • Free-roll tray (composer "+") used for routine rolls — it's the escape hatch, not the default. Most rolls flow through GM prompts in chat.
  • Speakers indistinguishable. GM / You / System must differ by border-style, alignment, AND label, not just color (grayscale wireframes don't have it).
  • Time-counter (50-baseline) hidden in a menu — must live in the app bar; it's the chapter-wide scarcity timer.
  • Roll result lost in the scroll — sticky "last roll" toast for ~3s after a roll resolves.
  • Casting a spell at a tier the caster hasn't unlocked silently fails — the in-chat Cast prompt must lock with explainer.
  • Boss-battle 0-HP state same as normal — must trigger a distinct overlay (Blaze of Glory · Risk It All).

Accessibility / focus

  • Focus order: app bar → vital strip / pull handle → most recent transcript card → in-chat roll controls (when present) → composer.
  • New transcript items use role="log" aria-live="polite"; rolls announce result text ("d20 14 plus 0 Wit equals 14, pass").
  • Pull-down handle is a real button with aria-label "Open character sheet"; activates on tap as well as drag for keyboard / switch-control users.
  • In-chat roll button is a single primary action; the Adv/Norm/Dis indicator next to it is a role="status" badge, not an interactive control. Adv/Dis state and reason are announced when the prompt appears.
  • Composer Send button is a real button with aria-label "Send action"; long-press surfaces secondary verbs.
  • Conditions are listed as text in the chip ("Prone"), not symbol-only.
  • Keyboard-ish: Esc closes overlays/tray, returns focus to the trigger; ↑/↓ moves between transcript cards.
  • Dense UI contrast: chip ink 1.5px stroke on paper ≥ 7:1; mono labels ≥ 4.5:1 at 12px+.

3 · Character Sheet — embedded in Session

Note: the sheet is no longer a separate screen. It's embedded in Session and revealed by pulling the stats row down (see Session A7 & A8). The mocks below document the sheet's content and tabs for engineering reference — not a standalone route.

Primary jobs

  • Look up a stat / ability / spell mid-session in < 5s without losing Story
  • See HP, mana, Evasion, conditions, XP-to-next at a glance
  • Read a class ability or spell description in full
  • Trigger a roll for an item / spell / saving throw without copy-paste
  • Confirm a level-up choice (HP / Evasion / Class ability) when XP threshold crossed

Always visible

  • Character header: name, class, level, XP bar
  • Stat block: Mig / Fin / Wit / Pre with modifiers
  • HP · Mana · Evasion summary row (this is the same stats row from Session)
  • Pull-handle for closing (drag up past mid-screen)

Swipe / tabs

  • Tabs: Combat · Spells · Inventory · Abilities · Notes (Mig/Fin/Wit/Pre row above tabs; elevated tab bar per theme)
  • Spell/ability detail opens as accordion expand, not a new route
  • "Roll from sheet" stays inline — uses the same roll tray as Session

Component inventory (rules-grounded)

  • Stat block: 4 stats; modifiers only (+2 / +1 / 0 / −1 starting layout shown explicitly)
  • HP / Mana / Evasion / Initiative chips with formula tooltip
  • Conditions drawer: Poisoned · Stunned · Frightened · Restrained · Prone (effect + removal)
  • Combat block: saving throws first (10 + stat), then weapons (stat, damage), armor (Light/Medium/Heavy w/ class restriction)
  • Abilities tab: starting ability + level-up picks (level/tier locked rows greyed)
  • Spells tab (Mage/Bard): tier columns Minor/Major/Mythic with cost 1/2/3; tier locks by class+level
  • Inventory: currency CP · SP · GP · PP (separate values); carried items; active weapon
  • Rest buttons: ☼ Short (day nap) / ☾ Long (overnight) — HP, MP, time-counter cost
  • Level-up modal: HP (Fixed (hd÷2)+Mig vs Roll hd+Mig) · Evasion +1 · Class ability

  • "Roll from sheet" opens a different roll system than Session — must reuse the same tray.
  • Level-up choice presented as a notification rather than a blocking state — XP threshold = level-up is automatic, the UI must enforce a choice before returning to Story.
  • Class restrictions on armor (Mage light only, etc.) shown only at character creation — surface as a lock on equip in inventory.
  • Abilities tab shows a flat list — must cluster by Lv 1 / 3 / 5 / 7 / 10 to mirror progression gates.

Accessibility / focus

  • Overlay traps focus; Esc / × returns focus to the trigger row in the Story.
  • Tab strip is a real tablist; arrow keys move tabs, Tab moves into panel.
  • Stat chips read as "Might plus 2", "Wit zero", "Presence minus 1" — not just numerals.
  • Roll buttons announce composition: "Attack: d20 plus Might plus 2, plus ability plus 1".
  • Level-up modal is role="dialog" aria-modal="true"; cannot be dismissed by background tap when XP threshold has been crossed.
  • Min tap target 44×44pt for every weapon/spell/ability row; row hit area extends full-width.
  • Use both fill AND label for mana pips ("●" filled, "○" empty + count "5/7").

SESSION

The button rolls in place and the result lands in the next System card. The Adv/Norm/Dis state is set by the engine (rules, conditions, the GM's call) — never a player toggle.
- Crit: nat-20 → maxed dice + rolled dice + mods once. Render visibly on the result card.

The handle also acts as a button (tap to open / close) for keyboard / switch-control users — aria-label "Open character sheet".

SHEET (embedded in Session — two-phase reveal)
- Drag the stats row down to expand. Phase 1 (~0..90px): the stats row translates down following the finger, revealing a Character Header row above it (name, class, level, XP bar). Phase 2 (~90..470px): char header and stats stay anchored; below the stats row the body reveals top-down (ability scores → vitals panel → segmented tabs → combat / abilities / spells / inventory panels).
- Threshold = finger past mid-screen at release → snap fully open; before mid-screen → snap back to resting (only stats row visible). Tapping the handle toggles. Dragging up past mid-screen while open → snap closed.
- Chat stays in flow underneath — it is pushed down (and shrinks) as the sheet expands. Never dimmed or covered.
- Char header: name, class, level, XP bar. No close (×) button — dismiss is the drag/tap on the handle.
- StatBar always visible (the stats row itself): HP / Mana (casters) / Eva / Init OR most-pressing condition chip. Max 4 chips at 320px.
- Ability scores row: Might / Finesse / Wit / Presence with sign-explicit modifiers (+2/+1/0/-1 starting layout when level 1).
- Tabs (elevated segmented bar inside sheet body): Combat · Spells (casters only) · Inventory · Abilities · Notes; Mig/Fin/Wit/Pre row fixed above tabs.
- Combat tab: weapon rows (stat tag, damage dice + stat); armor rows (Eva bonus, class restriction — Mage light only, Ranger/Bard light+medium, Warrior all); saves displayed as numeric (10 + stat).
- Abilities tab: clustered by level gate (1, 3, 5, 7, 10). Locked rows greyed with unlock requirement.
- Spells tab: tier filter (Minor 1MP · Major 2MP · Mythic 3MP); locked tiers visible but disabled. Mana shown as filled/empty pips with numeric "current/max".
- Inventory: currency CP · SP · GP · PP (separate values); carried items; active weapon flag.
- Rest icons: ☼ Short rest (day nap, t−1) · ☾ Long rest (overnight, t−5).
- Level-up state: blocking dialog when totalXP crosses threshold. Three choices: HP {Fixed (hd/2)+Mig | Roll hd+Mig} · Evasion +1 · Class ability (opens picker). Cannot resume Story until choice is confirmed. Outside-battle gate enforced by app.
